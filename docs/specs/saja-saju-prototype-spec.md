# 작업 명세: saja-saju 프로토타입 세팅 (수직 단면)

> Claude Code 작업 지시용. 새 빈 레포(saja-saju, private)에서 시작.
> 목적: "엔진(saju-engine)이 실제 웹앱 환경에서 import돼 정확한 명식을 낸다"를 확인하는
> 가장 얇은 수직 단면. 디자인·몰입 연출·결제·자동완성은 전부 범위 밖.

## 스택

- Next.js (App Router), TypeScript, Tailwind.
- 로컬에서 도는 것까지가 목표. 배포(Vercel)는 이번 범위 밖.

## 엔진 연결

- saju-engine을 git 의존성으로 설치. **커밋 해시 고정.**
  - `"saju-engine": "github:f7h6ts59d4-collab/saju-engine#<7eca3117eff8eaf2f5fd017e99ff06fc0e43fd2c>"`
- saju-engine은 private이므로 로컬 설치 시 GitHub 인증 필요(이미 로그인돼 있으면 됨).
  배포 인증은 이번 범위 밖.
- 엔진에서 `correctPillars`를 import해 사용한다.

## 입력 (폼 5개)

1. 생년월일 (양력/음력 선택)
2. 태어난 시간 (+ "시간 모름" 체크박스)
3. 출생지 (아래 "도시 처리" 참고)
4. 성별 (남/여)
5. 이름

### 계산에 쓰이는 입력 vs 표시용 입력 (중요)
- **correctPillars에 넘기는 것은 3개뿐**: 생년월일, 시간, 출생지(→IANA tz id).
- **성별·이름은 계산에 쓰지 않는다.** 받아서 화면에 그대로 표시만. (이후 LLM 개인화용 데이터)
- 성별·이름을 사주 계산 로직에 끼워 넣지 말 것.

### 도시 처리 (최소화)
- 자동완성 UI 없음. **국가/도시 몇 개를 IANA tz id와 함께 하드코딩한 드롭다운.**
  - 예: 서울=Asia/Seoul, 마닐라=Asia/Manila, 도쿄=Asia/Tokyo, 뉴욕=America/New_York,
    로스앤젤레스=America/Los_Angeles 정도 5개 내외.
- 선택한 도시 → tz id를 correctPillars에 전달. "도시 선택 → tz id 매핑 → 엔진 전달"이
  도는지 확인이 목적. 전 세계 자동완성은 다음 단계.

### "시간 모름" 처리
- 체크 시: 시간 입력을 비활성화하고, 정오(12:00) 가정으로 correctPillars 호출(timeUnknown=true).
- 결과의 시주에 "참고용(시간 미상)" 표시.

## 출력 (명식 표시)

- correctPillars 결과의 4기둥(연·월·일·시주)을 한글+한자로 표시.
- 입력한 성별·이름도 같이 표시(계산 안 쓰임을 확인하는 의미도 있음).
- 디자인 최소. 텍스트로 4기둥이 정확히 뜨면 충분. 몰입 연출·이미지 없음.

## 검증

- 로컬에서 `npm run dev`로 띄워, 알려진 케이스 입력 시 명식이 정확한지 수동 확인.
  - 예: 1990-05-15 14:30 서울 → 경오 신사 경진 계미
  - 예: 2024-02-04 17:26 서울(입춘 절입 전) → 연주 계묘 / 월주 을축
  - 예: 2024-02-04 17:28 서울(입춘 절입 후) → 연주 갑진 / 월주 병인
  - 예: 시간 모름 체크 → 시주에 "참고용" 표시, 나머지 정상
- typecheck / build 통과.
